Docker — платформа для упаковки приложений в контейнеры.
Docker — это программа, которая позволяет упаковать приложение со всем, что ему нужно для работы (библиотеки, настройки, зависимости), в изолированный «контейнер».
Зачем нужен: чтобы приложение гарантированно работало одинаково на любом компьютере — у разработчика, тестировщика, на сервере. Никаких проблем «у меня работает, а у тебя нет».

Главные преимущества:
1) Портативность — контейнер можно запустить где угодно.
2) Лёгкость — контейнеры не содержат целую операционную систему, как виртуальные машины, они занимают меньше места и запускаются быстрее.
3) Изоляция — приложения не конфликтуют между собой, даже если требуют разные версии библиотек.
4) Масштабирование — легко запустить много копий приложения.
5) Упрощение разработки — вся команда использует одинаковое окружение.

Контейнер — изолированное окружение с всем необходимым для работы приложения (код, зависимости).
Образ — неизменяемый шаблон, из которого создаются контейнеры (как чертёж).

>docker ps // Показывает список запущенных контейнеров.
>docker ps -a // Показывает все контейнеры, включая остановленные.

>docker run <имя_образа> // Запускает новый контейнер из указанного образа.
>docker stop <идентификатор_контейнера> // Останавливает запущенный контейнер.
>docker rm <идентификатор_контейнера> // Удаляет один или несколько остановленных контейнеров.
>docker start <идентификатор_контейнера> // Запускает ранее остановленный контейнер.

>docker run <image> — создать и запустить контейнер из образа.
>docker ps — список запущенных контейнеров (-a — все).
>docker images — список образов.
>docker pull <image> — скачать образ.
>docker build -t <name> . — собрать образ из Dockerfile.
>docker stop <container> — остановить контейнер.
>docker rm <container> — удалить контейнер.
>docker rmi <image> — удалить образ.
>docker exec -it <container> <cmd> — выполнить команду внутри работающего контейнера.
>docker logs <container> — посмотреть логи.
>docker system prune — очистить неиспользуемые ресурсы (контейнеры, образы, сети).
----------------------
Docker Compose — инструмент для запуска многоконтейнерных Docker-приложений, описанных в YAML-файле. Одной командой поднимаются все связанные сервисы.

>docker-compose down -v — останавливает и удаляет контейнеры, сети и тома (-v). Полная очистка.
>docker-compose build --no-cache — пересобирает образы с нуля, игнорируя кеш (полезно после изменений в Dockerfile).
>docker-compose up — создаёт и запускает контейнеры. Если добавить -d, то в фоновом режиме.
>docker-compose ps — список запущенных контейнеров.
>docker-compose logs [service] — логи сервиса (или всех).
>docker-compose stop — остановить контейнеры без удаления.
>docker-compose start — запустить остановленные.
>docker-compose restart — перезапустить.
>docker-compose exec <service> <command> — выполнить команду внутри работающего контейнера.
>docker-compose pull — скачать свежие образы для сервисов.
